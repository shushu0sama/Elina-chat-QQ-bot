# QQ 个人陪伴机器人 —「艾琳娜」

基于 NoneBot2 + DeepSeek V4 的 QQ 私聊 AI 伴侣，具备长期记忆、哲学知识库、交互式引导工具、主动聊天、内容推送和每日日记功能。

## 技术栈

| 层 | 技术 |
|---|---|
| 框架 | NoneBot2 (Python) |
| 协议 | OneBot V11 (对接 NapCatQQ) |
| 大模型 | DeepSeek V4-Pro（1.6T 参数，1M 上下文） |
| 分词 | jieba |
| 存储 | SQLite（支持多用户隔离） |
| 调度 | APScheduler（NoneBot 插件） |

## 功能一览

### 1. 个性化人格 + 状态系统

机器人叫「艾琳娜」，一个温柔但有自己性格的朋友。人格由 YAML 驱动，支持 6 种随机状态切换：

| 状态 | 权重 | 表现 |
|---|---|---|
| 正常 | 35% | 该怎样就怎样 |
| 话少/有点累 | 20% | 回复短，1-2 句话 |
| 话多/兴奋 | 15% | 多说几句，主动开话题 |
| 随意/摆烂 | 15% | 说废话开玩笑，用口语 |
| 温柔感性 | 10% | 更温柔，但不肉麻 |
| 有点烦 | 5% | 回复短促，不想营业 |

状态每次对话有 30% 概率重新 roll，模拟真实人类的情绪惯性。支持口语词（「害」「确实」「笑死」「好家伙」），允许不完美回复。

### 2. 哲学知识库

内置罗伯特·沙因费尔德《你值得过更好的生活》12 个核心概念的知识库：

**全息图 · 金钱游戏 · 第一阶段与第二阶段 · 赞赏感谢 · 流程工具 · 迷你流程 · 让话语充满力量 · 彩蛋 · 能量场 · 彻底解脱点 · 电影隐喻 · 大我**

每个概念含触发关键词、核心洞见、自然融入语、禁止事项和深入引导问题。LLM 根据用户消息自动检索最相关的 1-2 个概念，融入回复——不说教，像朋友之间自然流露的视角。

### 3. 交互式流程工具

当用户情绪低落时，不只是安慰——可以一步步带用户走书中的真实工具：

| 工具 | 触发 | 步数 |
|---|---|---|
| **完整流程** | 「陪我走流程」 | 6 步（正面迎击 → 彻底感受 → 说出真相 → 收回力量 → 绽放自己 → 赞赏感谢） |
| **迷你流程** | 「迷你流程」 | 3 步（看见 → 收回 → 感谢） |
| **赞赏感谢** | 「一起感谢」 | 3 层（感谢自己 → 感谢创造物 → 感谢过程） |

会话状态机管理，支持退出（「算了」「不做了」），30 分钟超时自动清理。

### 4. 长期记忆系统（多用户隔离）

六张 SQLite 表驱动，消息按 `user_id` 隔离：

| 表 | 用途 |
|---|---|
| `messages` | 完整对话记录（role + content + user_id + timestamp） |
| `summaries` | LLM 定期生成的对话摘要 |
| `key_memories` | 用户事实性信息：姓名、偏好、经历、习惯 |
| `active_users` | 活跃用户追踪 |
| `proactive_log` | 主动消息发送记录 |
| `key_memories` | 用户长期记忆 |

记忆流转：显式记忆（「记住：xxx」）→ 直接存入；自动提取（每 10 条消息 LLM 提取新事实，Jaccard 去重）；智能检索（jieba 分词 + 模糊匹配）；定期摘要（每 20 条消息触发 LLM 生成）。

### 5. 主动聊天 + 内容推送

两套独立的主动推送系统：

**对话式主动聊天：**

| 参数 | 默认值 | 说明 |
|---|---|---|
| `PROACTIVE_ENABLED` | true | 开关 |
| `PROACTIVE_INTERVAL_MINUTES` | 30 | 检查间隔 |
| `PROACTIVE_COOLDOWN_MINUTES` | 60 | 用户发言后多久不打扰 |
| `PROACTIVE_ACTIVE_HOURS_START` | 8 | 最早主动发言时间 |
| `PROACTIVE_ACTIVE_HOURS_END` | 24 | 最晚主动发言时间 |

**B 站内容推送：**

每 6 小时拉取 B 站热门/知识/科技区视频，用 jieba 匹配用户兴趣后用 LLM 精选推荐。

| 参数 | 默认值 | 说明 |
|---|---|---|
| `CONTENT_PUSH_ENABLED` | true | 开关 |
| `CONTENT_PUSH_INTERVAL_HOURS` | 6 | 检查间隔 |
| `CONTENT_PUSH_BILI_CATEGORIES` | 0,36,188 | B 站分区 RID |
| `CONTENT_PUSH_MAX_PER_PUSH` | 1 | 每次最多推几条 |

### 6. 每日日记

每天午夜 0:00，bot 自动将当天的对话整理成个人日记，保存为 Markdown 文件：

```
diaries/{user_id}/2026-05-13.md
```

每篇日记包含三个小节：**聊聊**（今日话题总结）/ **今日心情**（情绪变化推测）/ **艾琳娜的碎碎念**（艾琳娜视角的温柔观察）。当日消息不足 3 条自动跳过。

### 7. 多用户支持

所有消息和记忆按 `user_id` 隔离。多个 QQ 账号与 bot 私聊时，互相完全独立——每个人有自己的记忆、对话历史和日记。

### 8. 智能网页搜索（Function Calling）

当遇到不确定的事实、需要最新信息、或用户询问知识范围外的问题时，bot 会自动搜索互联网获取实时信息。

**工作流程：**

```
用户消息 → 拼装 prompt + 工具定义 → DeepSeek（带 function calling）
  ├─ LLM 自主判断需要搜索 → 提炼搜索 query
  │    → Bing CN 搜索（国内可用，无需 API Key）
  │    → 搜索结果格式化后喂回 LLM
  │    → 基于搜索结果生成自然回复
  └─ LLM 认为无需搜索 → 直接回复
```

**技术细节：**

| 方案 | 说明 |
|---|---|
| 触发机制 | DeepSeek Function Calling — LLM 自主决定要不要搜、搜什么 |
| 搜索后端 | 默认 Bing CN（`cn.bing.com`），国内直连，HTML 解析 |
| 备选后端 | DuckDuckGo（需境外网络）、可插拔架构支持扩展 |
| 容错 | Function calling 失败自动回退到普通对话，不影响正常聊天 |
| 深度阅读 | 支持 `fetch_page()` 获取搜索结果页面正文 |

**配置：**

| 参数 | 默认值 | 说明 |
|---|---|---|
| `WEB_SEARCH_ENABLED` | true | 开关 |
| `WEB_SEARCH_BACKEND` | bing | bing \| duckduckgo |
| `WEB_SEARCH_MAX_RESULTS` | 5 | 每次搜索结果数 |

### 9. 稳健工程



- **指数退避重试**：API 调用失败自动重试 3 次（1s → 2s → 4s）
- **错误分类提示**：区分频率限制、超时、认证失败，给出中文提示
- **消息智能分片**：按句子边界切割，适配 QQ 4000 字节限制
- **SQLite WAL 模式**：支持并发读写
- **会话超时保护**：流程工具会话 30 分钟自动清理

## 快速开始

### 1. 环境要求

- Python 3.11+
- NapCatQQ（已含在启动脚本中）

### 2. 配置

编辑 `.env` 文件，填入 DeepSeek API Key：

```env
DEEPSEEK_API_KEY=sk-your-key-here
DEEPSEEK_MODEL=deepseek-v4-pro
```

### 3. 启动

双击 `启动艾琳娜.bat`，自动启动 NapCat + Bot。

或者在终端：

```bash
venv\Scripts\python bot.py
```

### 4. 连接 QQ

首次使用需在 NapCatQQ 中配置反向 WebSocket 连接到 `ws://127.0.0.1:18080/onebot/v11/ws`（仅需配置一次）。

## 自定义

**切换人格**：编辑 `prompts/default.yaml`

**调整哲学知识**：编辑 `prompts/philosophy_knowledge.yaml`

**修改流程步骤**：编辑 `prompts/flow_steps.yaml`

所有改动无需改代码，重启即生效。
